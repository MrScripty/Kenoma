# Procedural Skin Graph Mesh Plan

Date: 2026-05-30
Status: Draft

## Goal

Build a simple modeling system where a user creates a low-detail character, mannequin, creature, prop, or organic shape by editing a graph of vertices and edges. The app generates a mesh around that graph.

This is a graph-to-surface modeling tool. The goal is an easy-to-implement and easy-to-use tool:

1. Start with one vertex.
2. Extrude vertices to create limbs, torso branches, fingers, horns, tails, props, or abstract forms.
3. Store a radius at every graph vertex.
4. Generate a surface around the graph edges and branch points.
5. Let the user keep editing the graph and regenerate the mesh interactively.

This plan is intentionally separate from:

- Fixed-topology morph-target systems.
- Detailed anatomy.
- Biomechanical simulation.
- Muscle/fat/skin layers.
- Production-quality automatic retopology.

## Runtime Contract

The implementation stack for this plan is the Rust workspace plus Bevy 0.18 for the editor/runtime viewer. The graph data model and mesh-generation core must remain framework-independent; Bevy integration must live behind the Bevy integration crate.

The editable `SkinGraph` asset and deterministic generation algorithm described in this plan are the source of truth. Generated meshes are disposable outputs for Bevy rendering and tests.

## Use Cases

- Fast mannequin blocking.
- Rough body or creature shape sketching.
- Simple armature-like visual scaffolds.
- Collision proxies.
- Control-volume previews for rigs.
- Low-detail generated characters for prototypes.
- Optional starting point before later sculpting, remeshing, or rigging tools.

## Graph Skinning Design

The input is a mesh-like graph made of vertices and edges. Every graph vertex has skin data containing radii and flags. The generator subdivides long edges, creates oriented frames around nodes, connects frames with quads, caps ends, and builds simple or advanced branch surfaces around branch vertices.

Useful concepts:

- Per-node radius controls.
- Root nodes for orienting frame propagation.
- Edge-frame propagation from roots.
- Generated quads along edges.
- End caps for terminal nodes.
- Special branch handling for vertices with more than two connected edges.
- Optional branch smoothing.
- Optional generated armature from the graph.

Concepts to simplify:

- Avoid full convex hull branch repair for MVP.
- Avoid complex topology cleanup at first.

## Non-Goals

- No accurate anatomy.
- No fixed-topology morph compatibility.
- No tissue simulation.
- No physically accurate volume preservation.
- No automatic high-quality quad retopology.
- No subdivision/evolution/fairing pass in the first version.

## Self-Contained Implementation Contract

Treat the sections below as the implementation contract for an original Rust system.

Coordinate conventions:

- Use a right-handed local model space.
- `+Y` is up.
- `+X` is character right.
- `+Z` is character forward.
- Store all graph positions in model space.
- Generate mesh positions in the same model space.
- Use meters as the conceptual unit, but the editor can display unitless values.

Mesh conventions:

- Generated mesh indices are triangles.
- Authoring code may think in quads, but output should triangulate every quad as `(a, b, c)` and `(a, c, d)`.
- Triangle winding is counter-clockwise when viewed from outside the surface.
- Normals are generated by accumulating area-weighted face normals and normalizing per vertex.
- UVs are optional for MVP. If generated, use edge distance as `u` and ring fraction as `v`.

Default constants:

```rust
const DEFAULT_RADIUS: f32 = 0.25;
const MIN_RADIUS: f32 = 0.01;
const DEFAULT_RING_SIDES: usize = 8;
const MIN_RING_SIDES: usize = 4;
const MAX_RING_SIDES: usize = 32;
const TARGET_SEGMENT_LENGTH_FACTOR: f32 = 1.25;
const MAX_EDGE_SEGMENTS: usize = 64;
const ZERO_EDGE_EPSILON: f32 = 1.0e-5;
const WORLD_UP: Vec3 = Vec3::Y;
const FALLBACK_NORMAL: Vec3 = Vec3::X;
```

Required behavior:

- If a graph island has no root, choose the lowest node index in that island as its generated root and report a warning.
- If a graph island has multiple roots, use all explicit roots as traversal seeds and report a warning.
- If an edge length is below `ZERO_EDGE_EPSILON`, skip mesh generation for that edge and report a warning.
- Clamp every radius to at least `MIN_RADIUS`.
- Regeneration must be deterministic for the same graph input.
- The graph database record is the saved asset. The generated mesh can always be rebuilt.
- Skin graphs are stored in the project `.human.sqlite` database in `skin_graphs` and related child tables.
- Do not persist generated mesh buffers as user project files. If a debug cache is ever added, key it by graph content hash and generation options, and make it safe to delete.

## Editing Workflow

Core user operations:

- Add node.
- Select node.
- Move node.
- Extrude selected node to create a new connected node.
- Connect two existing nodes.
- Delete selected node or edge.
- Scale node radius uniformly.
- Scale node radius on two local frame axes.
- Equalize radius axes.
- Mark selected node as root.
- Mark or clear loose branch behavior.
- Toggle generated mesh visibility.
- Toggle graph wire visibility.

Root editing rule:

- Marking a node as root clears the root flag from every other node in the same connected island.
- Other islands keep their own roots.
- If the graph is disconnected and a node is moved or deleted, validation rechecks roots per island.
- Loading may accept missing or multiple roots as recoverable warnings, but the editor command path should keep one explicit root per island whenever possible.

Practical shortcuts:

- `E`: extrude selected node.
- `G`: move selected node.
- `R`: rotate selected branch or local frame later.
- `S`: scale node radius.
- `X`: delete selected node or edge.

The MVP does not need an advanced DCC-style transform system. It needs direct Bevy gizmos, clickable nodes, and a small inspector panel.

## Mesh Generation Algorithm

### Step 1: Validate Graph

Checks:

- At least one node exists.
- Every edge uses valid nodes.
- No duplicate edges.
- No zero-length edges, or report and skip them.
- Every connected island has one root, or assign one automatically.
- Node radii are positive and clamped to a small minimum.

Validation pseudocode:

```rust
fn validate_graph(graph: &SkinGraph) -> GraphValidation {
    let mut warnings = Vec::new();
    let mut valid_edges = Vec::new();

    for (edge_id, edge) in graph.edges.iter().enumerate() {
        if edge.a >= graph.nodes.len() || edge.b >= graph.nodes.len() {
            warnings.push(InvalidEdgeIndex(edge_id));
            continue;
        }
        if edge.a == edge.b {
            warnings.push(SelfEdge(edge_id));
            continue;
        }
        let len = graph.nodes[edge.a].position.distance(graph.nodes[edge.b].position);
        if len < ZERO_EDGE_EPSILON {
            warnings.push(ZeroLengthEdge(edge_id));
            continue;
        }
        valid_edges.push(edge_id);
    }

    GraphValidation { valid_edges, warnings }
}
```

### Step 2: Build Connectivity

For each node, compute:

```text
degree[node] = number of connected edges
adjacent_edges[node] = edges touching node
```

Node classification:

- `Isolated`: degree 0
- `End`: degree 1
- `Connection`: degree 2
- `Branch`: degree 3 or more

### Step 2B: Generate Isolated Nodes

An isolated node has no incident edges, but it should still produce visible geometry so the user can start from a single point and edit radius before extrusion.

MVP isolated-node output:

1. Generate a low-poly ellipsoid centered at the node position.
2. Use `radius_x` for local X, `radius_y` for local Y, and `(radius_x + radius_y) * 0.5` for local Z.
3. Tag every generated vertex with `source_node = Some(node_id)`.
4. Generate smooth normals.
5. If generated rigging is enabled later, attach the ellipsoid to a helper bone or mark it as unriggable until it receives an edge.

Validation:

- An isolated graph with one node produces a nonempty mesh.
- Every generated triangle has three distinct vertex indices.
- Every generated vertex has finite position and normal.

### Step 3: Subdivide Long Edges

Long edges should get intermediate generated nodes so the surface bends more smoothly.

Simple rule:

```text
average_radius = (radius(a) + radius(b)) * 0.5
segment_count = ceil(edge_length / (average_radius * target_spacing_factor))
```

Clamp segment count to avoid runaway mesh generation:

```text
segment_count = clamp(segment_count, 1, 64)
```

Generated intermediate nodes interpolate:

- position
- radius_x
- radius_y
- source edge id

Subdivision pseudocode:

```rust
fn edge_segment_count(a: &SkinNode, b: &SkinNode) -> usize {
    let length = a.position.distance(b.position);
    let ra = (a.radius_x + a.radius_y) * 0.5;
    let rb = (b.radius_x + b.radius_y) * 0.5;
    let radius = ((ra + rb) * 0.5).max(MIN_RADIUS);
    let target_len = radius * TARGET_SEGMENT_LENGTH_FACTOR;
    ((length / target_len).ceil() as usize).clamp(1, MAX_EDGE_SEGMENTS)
}

fn interpolate_node(a: &SkinNode, b: &SkinNode, t: f32) -> DerivedNode {
    DerivedNode {
        original: None,
        source_edge: Some(edge_id),
        position: a.position.lerp(b.position, t),
        radius_x: lerp(a.radius_x, b.radius_x, t).max(MIN_RADIUS),
        radius_y: lerp(a.radius_y, b.radius_y, t).max(MIN_RADIUS),
        degree: 2,
    }
}
```

Optional taper-aware interpolation:

```rust
fn radius_biased_t(a_radius: f32, b_radius: f32, t: f32) -> f32 {
    let ratio = (b_radius.max(MIN_RADIUS) / a_radius.max(MIN_RADIUS)).clamp(0.25, 4.0);
    let exponent = ((ratio + 1.0) * 0.5).clamp(0.5, 2.0);
    t.powf(exponent)
}
```

Use this only for generated intermediate nodes, not for source node positions. It keeps tapering limbs from placing too many visible segments near one endpoint while preserving deterministic output.

### Step 4: Build Edge Frames

Each generated edge segment needs an orientation frame:

```text
tangent = normalize(b.position - a.position)
normal = stable_perpendicular(tangent, previous_normal_or_world_up)
binormal = cross(tangent, normal)
```

For chains, propagate frame orientation along the graph from the root so frames do not twist randomly.

Simple MVP approach:

1. Start at each island root.
2. Pick an initial normal from world up unless the edge is nearly parallel.
3. Traverse edges depth-first or breadth-first.
4. For each child edge, parallel-transport the previous frame onto the new tangent.
5. At branches, start each outgoing edge with the parent frame adjusted to the outgoing tangent.

This avoids most visible twisting without needing any external edge-frame implementation.

Frame utilities:

```rust
fn stable_perpendicular(tangent: Vec3) -> Vec3 {
    let up_dot = tangent.dot(WORLD_UP).abs();
    let seed = if up_dot > 0.95 { FALLBACK_NORMAL } else { WORLD_UP };
    seed.cross(tangent).normalize()
}

fn make_frame(tangent: Vec3, previous_normal: Option<Vec3>) -> Frame {
    let tangent = tangent.normalize();
    let normal = match previous_normal {
        None => stable_perpendicular(tangent),
        Some(prev) => {
            let projected = prev - tangent * prev.dot(tangent);
            if projected.length_squared() < 1.0e-8 {
                stable_perpendicular(tangent)
            } else {
                projected.normalize()
            }
        }
    };
    let binormal = tangent.cross(normal).normalize();
    Frame { tangent, normal, binormal }
}
```

Traversal rule:

```text
for each island:
    queue roots
    for every outgoing edge from a root:
        create frame from edge tangent and no previous normal
        push neighbor with that frame
    while queue is not empty:
        pop node and incoming frame
        for each unvisited outgoing edge:
            create child frame from child tangent and incoming normal
            push child node with child frame
```

Loops are supported by marking edges visited. When a loop edge is reached after both sides already have frames, use the frame from the first traversal path and report `LoopFrameApproximated`.

If a loop or two-way traversal creates a visible frame seam, the MVP should keep the first valid frame, mark the seam edge in diagnostics, and choose the bridge order that minimizes total squared edge length between ring vertices. The advanced loop-twist correction can later distribute the seam around the loop, but the MVP must never generate NaN frames or duplicate triangle indices when a loop is present.

### Step 5: Create Rings

At every generated node that is not a branch, create a ring around the graph node.

MVP ring resolution:

```text
ring_sides = 8
```

This is simple and gives smoother mannequin tubes than square frames. Later the user can choose 4, 6, 8, 12, or 16 sides.

Ring point:

```text
angle = side_index / ring_sides * TAU
point = position
      + normal * cos(angle) * radius_x
      + binormal * sin(angle) * radius_y
```

Ring generation pseudocode:

```rust
fn add_ring(mesh: &mut MeshBuilder, node: &DerivedNode, frame: Frame, sides: usize) -> Ring {
    let sides = sides.clamp(MIN_RING_SIDES, MAX_RING_SIDES);
    let mut indices = Vec::with_capacity(sides);
    for side in 0..sides {
        let angle = side as f32 / sides as f32 * std::f32::consts::TAU;
        let p = node.position
            + frame.normal * angle.cos() * node.radius_x.max(MIN_RADIUS)
            + frame.binormal * angle.sin() * node.radius_y.max(MIN_RADIUS);
        indices.push(mesh.push_vertex(p));
    }
    Ring { indices }
}
```

### Step 6: Connect Rings Along Edges

For each generated edge segment with ring `A` and ring `B`, create quads:

```text
for side in 0..ring_sides {
    a0 = ring_a[side]
    a1 = ring_a[(side + 1) % ring_sides]
    b0 = ring_b[side]
    b1 = ring_b[(side + 1) % ring_sides]
    add_quad(a0, b0, b1, a1)
}
```

Normals can be computed from faces at first, then averaged per vertex for smooth shading.

Use this exact winding for an edge whose frame tangent points from ring `A` to ring `B`:

```rust
fn connect_rings(mesh: &mut MeshBuilder, a: &Ring, b: &Ring) {
    let sides = a.indices.len();
    assert_eq!(sides, b.indices.len());
    for side in 0..sides {
        let a0 = a.indices[side];
        let a1 = a.indices[(side + 1) % sides];
        let b0 = b.indices[side];
        let b1 = b.indices[(side + 1) % sides];
        mesh.add_quad(a0, b0, b1, a1);
    }
}

fn add_quad(mesh: &mut MeshBuilder, a: u32, b: u32, c: u32, d: u32) {
    mesh.indices.extend_from_slice(&[a, b, c, a, c, d]);
}
```

### Step 7: Cap Ends

For degree-1 end nodes:

MVP cap options:

- Flat cap: one center vertex plus triangles to the ring.
- Rounded cap later: hemisphere-like rings.

Use flat caps first because they are predictable and easy to test.

Cap winding:

```rust
fn cap_ring(mesh: &mut MeshBuilder, ring: &Ring, center: Vec3, outward: Vec3) {
    let c = mesh.push_vertex(center);
    let sides = ring.indices.len();
    for side in 0..sides {
        let v0 = ring.indices[side];
        let v1 = ring.indices[(side + 1) % sides];
        if face_normal_points_out(center, mesh.position(v0), mesh.position(v1), outward) {
            mesh.indices.extend_from_slice(&[c, v0, v1]);
        } else {
            mesh.indices.extend_from_slice(&[c, v1, v0]);
        }
    }
}
```

For an end node, `outward` is `-edge_tangent` at the start of a chain and `edge_tangent` at the end of a chain.

### Step 8: Handle Branches

Branch nodes are the hard part. Keep MVP branch generation simple.

MVP branch strategy:

1. Create one sphere-like or ellipsoid hub at the branch node.
2. For every incident edge, create a ring slightly away from the branch center.
3. Connect each edge ring to the hub with triangles or quads.
4. Accept that branch topology is not perfect.

Practical hub implementation:

- Generate a UV sphere or low-poly ico sphere scaled by `radius_x`, `radius_y`, and an averaged radius.
- For each incident edge, connect the first edge ring to nearby hub vertices, or simply allow controlled overlap for MVP.

MVP branch implementation should use controlled overlap because it is robust and fast:

1. Generate a low-poly UV sphere at the branch node.
2. Do not boolean-union it with outgoing tubes.
3. Start each outgoing edge ring at `branch.position + tangent * branch_radius * 0.75`.
4. Let the tube penetrate the hub slightly.
5. Smooth normals independently for the hub and tubes.

This produces a usable mannequin even when topology is not mathematically manifold. Report `BranchFallbackOverlap` in diagnostics so later tooling can identify these regions.

Branch safety rules:

- Never emit a triangle with duplicate vertex indices.
- If two generated branch vertices are closer than `GRAPH_POSITION_EPSILON`, keep both unless an explicit merge pass owns the topology update.
- Hub vertices must be tagged `BranchGenerated`.
- Tube ring vertices adjacent to the branch must keep their `source_edge` provenance and must not be moved by branch smoothing more than `BRANCH_SMOOTH_MAX_RING_MOVE_FRACTION` of the local radius.
- Branch fallback overlap is acceptable for MVP, but it must be deterministic and must report the branch node id.

Low-poly hub generation:

```rust
fn add_branch_hub(mesh: &mut MeshBuilder, node: &DerivedNode, rings: usize, sides: usize) {
    let ry = node.radius_y.max(MIN_RADIUS);
    let rx = node.radius_x.max(MIN_RADIUS);
    let rz = ((rx + ry) * 0.5).max(MIN_RADIUS);

    for lat in 0..=rings {
        let v = lat as f32 / rings as f32;
        let phi = -std::f32::consts::FRAC_PI_2 + v * std::f32::consts::PI;
        for lon in 0..sides {
            let u = lon as f32 / sides as f32;
            let theta = u * std::f32::consts::TAU;
            let p = node.position + Vec3::new(
                theta.cos() * phi.cos() * rx,
                phi.sin() * ry,
                theta.sin() * phi.cos() * rz,
            );
            mesh.push_vertex(p);
        }
    }

    // Connect latitude/longitude grid with triangles, skipping degenerate pole quads.
}
```

Better branch strategy for later:

1. Build rings for every incident edge.
2. Collect all ring vertices.
3. Build a convex hull around those rings and the branch radius.
4. Merge or bridge the hull to outgoing edge rings.
5. Smooth the branch area.

This should be delayed until the simple hub is not good enough.

### Step 9: Smooth Mesh

MVP:

- Smooth normals only.

Later:

- Optional Laplacian smoothing around branch hubs.
- Do not smooth end rings or selected hard nodes unless requested.
- Store original generated positions so smoothing can be blended.

Normal generation:

```rust
fn recompute_normals(mesh: &mut GeneratedSkinMesh) {
    mesh.normals.clear();
    mesh.normals.resize(mesh.positions.len(), Vec3::ZERO);
    for tri in mesh.indices.chunks_exact(3) {
        let a = tri[0] as usize;
        let b = tri[1] as usize;
        let c = tri[2] as usize;
        let normal = (mesh.positions[b] - mesh.positions[a])
            .cross(mesh.positions[c] - mesh.positions[a]);
        mesh.normals[a] += normal;
        mesh.normals[b] += normal;
        mesh.normals[c] += normal;
    }
    for normal in &mut mesh.normals {
        *normal = if normal.length_squared() > 1.0e-12 {
            normal.normalize()
        } else {
            Vec3::Y
        };
    }
}
```

## Advanced Self-Contained Skin Graph Specs

These features are not required for the first useful prototype. They define enough behavior to implement higher-quality procedural graph modeling.

### Twist-Stable Frame Propagation For Loops

The MVP uses first-visited frame propagation. For loops, this can create a visible twist discontinuity where traversal paths meet. The advanced implementation distributes twist error around each closed loop.

Loop detection:

1. During graph traversal, when visiting an already visited node through an unvisited edge, record a loop.
2. Recover the two existing tree paths from the shared ancestor to the loop endpoints.
3. Concatenate path A, the new edge, and reversed path B to form an ordered loop edge list.

Frame correction:

```rust
fn distribute_loop_twist(loop_rings: &[RingId], frames: &mut [Frame]) {
    let first = frames[loop_rings[0].0];
    let last = frames[*loop_rings.last().unwrap().0];
    let twist = signed_angle_around_axis(last.normal, first.normal, first.tangent);
    let n = loop_rings.len().max(1) as f32;
    for (i, ring) in loop_rings.iter().enumerate() {
        let correction = -twist * (i as f32 / n);
        frames[ring.0].normal = rotate_around_axis(
            frames[ring.0].normal,
            frames[ring.0].tangent,
            correction,
        );
        frames[ring.0].binormal = frames[ring.0].tangent.cross(frames[ring.0].normal).normalize();
    }
}
```

Validation:

- A circular graph closes with first and last ring normals within `5` degrees.
- A figure-eight graph reports two loop corrections.
- Open chains are unchanged by loop correction.

### Manifold Branch Generation

The MVP branch hub can overlap outgoing tubes. Advanced branch generation should create connected topology.

Input:

- Branch center `C`.
- Incident edge tangents `T[i]`.
- Incident edge rings `R[i]`, each with `ring_sides` vertices.
- Branch radius `r`.

Algorithm:

1. Move each incident ring away from the branch center:

```text
ring_center[i] = C + T[i] * r * 0.75
```

2. Create one branch center vertex at `C`.
3. For each incident ring, create a fan from `C` to that ring. This is guaranteed connected but can look pinched.
4. If the angle between two adjacent incident tangents is less than `130` degrees, bridge nearest ring side spans between those two rings.
5. Smooth only vertices tagged `BranchGenerated`.

Ordering incident edges:

```rust
fn sort_branch_edges(tangents: &mut [Vec3], seed_normal: Vec3) {
    let axis = average(tangents).normalize_or(Vec3::Y);
    let ref_n = (seed_normal - axis * seed_normal.dot(axis)).normalize_or(Vec3::X);
    tangents.sort_by(|a, b| {
        let aa = signed_angle_around_axis(ref_n, *a, axis);
        let bb = signed_angle_around_axis(ref_n, *b, axis);
        aa.partial_cmp(&bb).unwrap()
    });
}
```

Bridge selection:

```rust
fn nearest_ring_side_pair(a: &Ring, b: &Ring, positions: &[Vec3]) -> (usize, usize) {
    let mut best = (0, 0);
    let mut best_dist = f32::MAX;
    for i in 0..a.indices.len() {
        for j in 0..b.indices.len() {
            let d = positions[a.indices[i] as usize].distance_squared(positions[b.indices[j] as usize]);
            if d < best_dist {
                best = (i, j);
                best_dist = d;
            }
        }
    }
    best
}
```

Acceptance criteria:

- A T branch produces one connected mesh component.
- No generated triangle has duplicate vertex indices.
- Branch smoothing does not move edge ring vertices more than `25%` of local radius.

### Branch Smoothing

Use constrained Laplacian smoothing on generated branch vertices.

```rust
fn smooth_branch_vertices(mesh: &mut GeneratedSkinMesh, tags: &[VertexTag], iterations: usize, alpha: f32) {
    let adjacency = build_vertex_adjacency(&mesh.indices, mesh.positions.len());
    for _ in 0..iterations {
        let old = mesh.positions.clone();
        for v in 0..mesh.positions.len() {
            if tags[v] != VertexTag::BranchGenerated {
                continue;
            }
            let neighbors = &adjacency[v];
            if neighbors.is_empty() {
                continue;
            }
            let avg = neighbors.iter().map(|n| old[*n]).sum::<Vec3>() / neighbors.len() as f32;
            mesh.positions[v] = old[v].lerp(avg, alpha.clamp(0.0, 1.0));
        }
    }
    recompute_normals(mesh);
}
```

Defaults:

```rust
const BRANCH_SMOOTH_ITERATIONS: usize = 3;
const BRANCH_SMOOTH_ALPHA: f32 = 0.35;
```

### Rounded End Caps

Flat caps are enough for MVP. Rounded caps improve mannequin quality.

Algorithm:

1. For an end ring with center `C`, tangent `T`, normal `N`, and binormal `B`, generate `cap_rings` smaller rings toward the tip.
2. Tip position is `C + outward * average_radius`.
3. Ring radius scales by `cos(theta)` where `theta` goes from `0` to `PI/2`.
4. Ring center moves by `sin(theta) * average_radius`.

```rust
fn add_rounded_cap(mesh: &mut MeshBuilder, base: &Ring, frame: Frame, center: Vec3, outward: Vec3) {
    let cap_rings = 3;
    let sides = base.indices.len();
    let mut previous = base.clone();
    for r in 1..=cap_rings {
        let t = r as f32 / (cap_rings + 1) as f32;
        let theta = t * std::f32::consts::FRAC_PI_2;
        let scale = theta.cos();
        let offset = theta.sin();
        let cap_center = center + outward * offset * average_radius_for_ring(base, mesh);
        let next = add_scaled_ring(mesh, cap_center, frame, scale, sides);
        connect_rings(mesh, &previous, &next);
        previous = next;
    }
    add_tip_fan(mesh, &previous, center + outward * average_radius_for_ring(base, mesh), outward);
}
```

### Generated Rig And Weights

Purpose:

- Let graph-generated mannequins be posed without hand rigging.

Rig generation:

1. Pick one root per graph island.
2. Traverse graph edges from root.
3. Create one bone per graph edge.
4. Bone head is the parent node position.
5. Bone tail is the child node position.
6. Parent bone is the edge that reached the parent node.
7. If an edge is part of a loop and both endpoints already have parent bones, mark it as `non_deforming_helper` or skip it for rig generation.
8. If an island contains only one isolated node, create a non-deforming helper bone with a small tail offset or mark the island as `NoRigForIsolatedNode`.

```rust
struct GeneratedBone {
    name: String,
    parent: Option<BoneId>,
    source_edge: EdgeId,
    head: Vec3,
    tail: Vec3,
}
```

Weight generation:

- Ring vertices generated along an edge get weight `1.0` for that edge's bone.
- Vertices near a graph node blend adjacent edge bones.
- Branch hub vertices blend all incident edge bones by inverse distance to each edge segment.
- Isolated-node vertices attach to the island helper bone if one exists; otherwise they remain static generated mesh vertices without skin weights.

Inverse-distance branch weighting:

```rust
fn branch_vertex_weights(p: Vec3, incident_bones: &[BoneSegment]) -> Vec<(BoneId, f32)> {
    let mut weights = Vec::new();
    for bone in incident_bones {
        let d = point_segment_distance(p, bone.head, bone.tail).max(0.001);
        weights.push((bone.id, 1.0 / (d * d)));
    }
    normalize_weights(&mut weights);
    keep_top4(&mut weights);
    weights
}
```

Validation:

- Every generated mesh vertex has at least one weight.
- Every deforming generated bone has at least one vertex.
- Rest-pose skinning returns the generated mesh unchanged.

### Procedural UVs

MVP can skip UVs. Advanced runtime mesh generation should generate predictable UVs when Bevy materials need them.

Tube UVs:

```text
u = cumulative_distance_along_edge_chain / uv_length_scale
v = side_index / ring_sides
```

Caps:

```text
u = 0.5 + local_x / (2 * radius_x)
v = 0.5 + local_z / (2 * radius_y)
```

Branches:

- Use spherical projection for branch hubs.
- Tag branch UV seams along the least visible incident edge, chosen as the edge pointing most backward in `-Z`.

Validation:

- Every generated vertex has one UV.
- UV coordinates are finite.
- Adjacent tube rings have monotonically increasing `u`.

## Risks

- Branch topology can become messy quickly.
- Frame propagation can twist unexpectedly on loops.
- Generated topology is not stable enough for fixed-topology morphs.
- Low ring resolution can look blocky.
- High ring resolution can generate too much geometry during interactive editing.
- User expectations may drift toward sculpting or retopology; keep MVP focused on fast shape blocking.
